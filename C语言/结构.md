## 结构

### 结构声明

<code>struct <strong>tag { <em>member-list</em> } <em>variable-list</em></strong></code>。

```c
struct {
    int a;
    char b;
    float c;
} x;
```

```c
struct {
    int a;
    char b;
    float c;
} y[20], *z;
```

> 这两个声明被编译器当作两种截然不同的类型， 即使它们的**成员列表**完全相同。

### 不完整声明

```c
struct B;

struct A {
    struct B *partner;
    /* other declarations */
};

struct B {
    struct A *partner;
    /* other declarations */
};
```

> 不完整声明，可以用在**不需要**知道这个结构的长度的声明中，如声明指向这个结构的指针。

### 结构的存储分配

> 结构的成员应该根据它们的边界需要进行重排，减少因边界对齐而造成的内存损失。

**`offsetof` 宏（`stddef.h`）**：`offsetof(type, member)` 表达式的结果是一个 `size_t` 值，表示这个指定成员开始存储的位置距离结构开始存储的位置偏移几个字节。

### 位段

```c
struct CHAR {
    unsigned ch		: 7;
    unsigned font	: 6;
    unsigned size	: 19;
};
struct CHAR ch1;
```

- 位段成员必须声明为 `int`、`signed int` 或 `unsigned int` 类型。
- 在成员名后面是一个冒号和一个整数，这个整数指定该位段所占用的位的数目。

> 应该用 `signed` 或 `unsigned` 整数显式地声明位段，因为如果把位段声明为 `int` 类型，它究竟被解释为有符号数还是无符号数是由编译器决定的。
>
> 注重可移植性的程序应该避免使用位段。
