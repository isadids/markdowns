## 使用的操作系统

- **Windows**：
  - **生态庞大**，有些事情不可避免的会用到 Windows；
  - **解决方案成熟**，遇到问题容易找到解决方法。
- **Linux**：
  - **开源免费**，**隐私性强**，**简洁**，**高性能**等；
  - **设计理念**，**代码架构**等本身就是很好的开发学习材料；
  - 一名*黑客*应该使用的操作系统。
- **MacOS**：
  - **办公体验强**；
  - **UI 设计间接美观**；
  - **基于 Unix**，性能优于 Windows。

## 目前的多系统混合模式

### ✅️主力系统 + 虚拟机

主力系统 Windows + 虚拟机 Linux。

- 由于 MacOS 的生态限制，Linux 极致的自由而导致的不稳定所以使用 Windows 作为主力系统来**托底**。
- MacOS 主要用于**办公**，其他领域（如游戏）生态不足，因此不能用作主力系统，只能当做领域专用系统。

### ❌️双系统

双系统切换麻烦都是小事，最大的痛点是不能**同时**运行以实现某些工作场景，比如运维测试。

## 具体解决方案

除了少数依赖 Windows 生态的领域，其他领域（开发，办公）应尽可能得转向 Linux，这就不可避免地要将“折腾”的重心转向 Linux，所以 Windows 系统应力求稳定。

### 主力系统版本的选择

最新**大版本**的 LTSC 版本（比如 *Windows 11 2024 LTSC*）：
  
LTCS 版本**更稳定**，且**更精简**，这样就能够将更多的**硬件资源**用于 Linux 虚拟机。

### 虚拟机平台软件的选择

✅️VMware Workstation Pro：

- **稳定易用**与**高性能**，尤其对虚拟机操作系统依赖更强，则更需要**稳定**与**高性能**。

❌️VirtualBox：

- 稳定性差，虽然是开源，但确实不如 VMware。

## Linux 系统发行版的选择

✅️Arch Linux：

- **可定制性**更强，高度自定义，适合用来学习 Linux 系原理。
- 个人理由：最开始在 Windows 上安装 MinGW-w64 时，经索发现使用 MSYS2 来安装是最好的方法之一，而 MSYS2 中用的 Linux 包管理工具是 **pacman**，而 Arch Linux 是使用 **pacman** 作为默认包管理工具的 Linux 发行版一。
